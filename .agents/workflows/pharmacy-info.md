1. T (Task, 작업):

가이드라인 요약의 품질 표준 설정 및 정확도 중심의 데이터 고도화
2. C (Context, 맥락):

약학 정보의 특성상 잘못된 정보는 치명적인 결과를 초래할 수 있음. 따라서 단순한 요약을 넘어, 검증된 우수 사례(Reference)와 동일한 수준의 깊이 있는 정보를 제공함으로써 플랫폼의 신뢰성을 확보하고자 함.
3. R (Requirement, 요구사항):

품질 벤치마킹 (Quality Standard):
세부 가이드라인: g252 항목의 요약 분량과 정보 밀도를 표준으로 삼아 작성할 것. (상세 기전, 임상 수치, 금기 사항 등 포함)
통합 가이드라인: 기존 '고혈압 가이드라인 요약'의 구조와 깊이를 표준으로 삼을 것. (학회별 비교 분석 및 핵심 결론 포함)
원칙: **속도보다 정확성(Accuracy over Speed)**을 절대적인 최우선 순위로 둔다. 빠르게 만드는 것보다 근거 중심의 정확한 데이터를 생성하는 데 집중할 것.
지속 적용: 이 기준은 일회성이 아니며, 가이드라인을 요약하는 모든 프로세스에서 항상 상기하고 준수해야 할 '대원칙'임.
4. E (Example, 예시):

세부 가이드라인 표준: g252 (약물군별 특징이 아주 상세하게 서술된 형태)
통합 가이드라인 표준: 고혈압 가이드라인 요약 (여러 학회의 지침을 표나 리스트로 완벽히 통합한 형태)
5. I (Instruction, 지시사항):

새로운 가이드라인 요약에 착수하기 전, 반드시 g252와 '고혈압 요약본'을 다시 읽고 해당 수준의 퀄리티를 리마인드한다.
데이터 추출 시 원문과 대조하여 수치나 약물명에 오류가 없는지 이중으로 체크한다.
작성된 초안이 설정된 표준(g252 등)에 비해 내용이 빈약하거나 깊이가 부족하지 않은지 스스로 검증한다.
내용의 정확성과 분량 기준이 모두 충족되었을 때만 최종 데이터로 승인하고 DB에 업로드한다.
